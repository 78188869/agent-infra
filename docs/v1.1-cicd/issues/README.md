# Issues

本目录存储版本开发过程中的 Issue 摘要和跟踪。

## Issue 文件命名

```
issue-{number}-{short-title}.md
```

例如：`issue-001-auth-timeout.md`

## Issue 模板结构

```markdown
# Issue #{number}: {Title}

## Summary
<!-- Issue 摘要 -->

## Impact
<!-- 影响范围 -->

## Status
{Open | In Progress | Resolved | Closed}

## Related
- PRD: [链接]
- TRD: [链接]
- ADR: [链接]

## Resolution
<!-- 解决方案（如果已解决） -->
```
