# Plans

本目录存储版本开发过程中的执行计划和任务跟踪。

## Plan 文件命名

```
{YYYY-MM-DD}-{short-title}.md
```

例如：`2026-03-23-api-implementation.md`

## Plan 模板结构

```markdown
# Plan: {Title}

## Context
<!-- 计划背景 -->

## Objectives
1. {目标1}
2. {目标2}

## Tasks

### Phase 1: {Phase Name}
- [ ] {Task 1}
- [ ] {Task 2}

### Phase 2: {Phase Name}
- [ ] {Task 1}
- [ ] {Task 2}

## Dependencies
<!-- 依赖项 -->

## Risks
<!-- 风险项 -->

## Status
{Not Started | In Progress | Completed | Blocked}
```
